# Kubernetes Deployment Strategies Project

This project demonstrates how to deploy and manage a containerized application in Kubernetes using both Rolling Update and Recreate deployment strategies.

Features

* Dockerized Student Management Application
* Push application images to Docker Hub
* Kubernetes Deployment with RollingUpdate strategy
* Kubernetes Deployment with Recreate strategy
* Service exposure using NodePort
* Application version upgrades
* Deployment rollback
* Rollout history tracking

Project Flow

Version 1 → Deploy to Kubernetes

Version 2 → Rolling Update (Zero Downtime)

Rollback → Restore Previous Version

Version 3 → Recreate Strategy (With Downtime)

Rollback → Restore Previous Stable Version

Technologies

* Docker
* Docker Hub
* Kubernetes
* NGINX
* GitHub

Learning Outcomes

* Kubernetes Deployments
* Rolling Updates
* Recreate Strategy
* Rollbacks
* Deployment History
* Service Management
* Container Image Versioning

This project is designed for DevOps engineers who want hands-on experience with Kubernetes deployment strategies.
